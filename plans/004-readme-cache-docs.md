## Current state

Behavior facts to document (verify each against the live code first):

1. **Cache location** (`src/go/DirectoryExtensions.cs:39-48`): caches live
   under `<temp>/dotnet/go/`, where `<temp>` is the user temp folder on
   Windows (`%TEMP%`) and the local application data folder on Linux/macOS
   (`~/.local/share` / `~/Library/Application Support`). One subdirectory per
   entry-point file.
2. **Per-app clean** (`src/go/Program.cs`, `clean` command): `go clean app.cs`
   deletes the cached publish artifacts for that file. Next run rebuilds.
3. **Clean everything** (added by plan 003): `go clean --all` deletes all
   cached apps.
4. **Automatic cleanup** (`src/go/CleanupScheduler.cs`,
   `src/go/CleanupManager.cs:5`): at most once every 2 days, after running
   your app, `go` spawns a detached background process that deletes cache
   directories not used for more than 2 days. Every run "touches" the app's
   cache directory (`src/go/Program.cs:85`), so actively used apps are never
   cleaned up.

## Cache and cleaning

`go#` caches build and publish outputs per entry-point file under the
user's temp area (`%TEMP%\dotnet\go` on Windows,
`~/.local/share/dotnet/go` on Linux, `~/Library/Application Support/dotnet/go` on macOS), which is what makes
unchanged re-runs near-instant.

```console
# Delete the cached artifacts for a single app (next run rebuilds)
dnx go clean app.cs

# Delete the cached artifacts for all apps
dnx go clean --all
```

Caches are also cleaned automatically: at most once every couple of days,
`go#` removes cache directories that haven't been used for a while, in a
detached background process. Apps you run regularly are never affected.
```
